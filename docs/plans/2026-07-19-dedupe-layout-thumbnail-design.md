# 去重页布局与缩略图预览设计

## 目标

- 去重组充分使用横向空间，竖卡和宽卡按实际宽度排列。
- 任意未填满的重复组行、组内卡片行都保持横向居中。
- 卡片附属的体积和页数信息始终相对卡片居中。
- 阅读器缩略面板从本次触发侧进入，不跨屏飞行。
- 去重页提供独立、精简的右键菜单和中央缩略图预览。

## 去重布局

外层重复组与内层归档卡片都使用可换行的弹性布局，不再使用固定 `316px` 栅格轨道。

- 竖向卡片保持现有约 `150px` 宽度。
- 宽幅卡片保持现有约 `316px` 宽度。
- 卡片项以归档卡片的实际宽度作为自身宽度；体积/页数胶囊在该宽度内居中。
- 组内优先在同一行容纳更多卡片，空间不足才换行。
- 重复组按内容所需宽度参与外层换行；每一行均使用 `justify-content: center`，包括中间行和最后一行。
- 小屏下宽卡允许收缩至容器宽度，避免横向溢出。

## 去重专用右键菜单

不复用首页包含删除、下载等操作的完整菜单，新增职责单一的去重菜单，仅包含：

1. 在新标签页打开阅读页。
2. 查看缩略图。

菜单沿用现有玻璃质感、视口边界约束、点击外部/滚动/Escape 关闭行为。右键菜单不会改变当前删除选择。

## 中央缩略图面板

采用与首页设置面板相同的中央遮罩和浮层结构，内部包含两级视图：

- 网格视图：加载归档文件清单，并按需加载所有页面缩略图。
- 大图视图：点击缩略图后在同一浮层显示对应页面原图；提供上一页、下一页、返回网格和关闭。

面板打开、翻页和关闭都不写入阅读历史或服务器阅读进度。缩略图继续复用现有图片缓存与 Minion 任务等待逻辑；原图沿用现有鉴权图片加载逻辑，关闭或换页时释放临时资源。

失败状态分别显示文件清单加载失败、缩略图失败或原图失败，并提供重试，不让面板永久停留在加载状态。

## 阅读器侧栏动画

将“目标侧”和“当前渲染侧”分开管理：

- 已关闭时，从目标侧的隐藏位置挂载，下一动画帧进入。
- 已打开且切换侧时，当前侧先完成退场，再切换渲染侧，下一动画帧从新侧进入。
- 动画期间屏蔽重复触发，关闭操作仍可立即响应。
- `prefers-reduced-motion` 下直接切换，不强制播放位移动画。

## 测试与验收

- 单测覆盖归档宽度分类、行布局所需宽度和新标签页 URL。
- 回归契约覆盖去重专用菜单、中央两级预览、缩略侧栏分阶段切换。
- 桌面窄/宽视口检查竖卡、宽卡、混合卡片与单个重复组。
- 运行 `npm test`、`npm run lint`、`npm run check`、`npm run build`。

## 自审结论

- 不修改去重算法、互锁规则、保存格式或删除流程。
- 不扩展现有全功能右键菜单，避免条件分支继续膨胀。
- 中央预览只共享底层图片加载能力，不共享阅读器页面状态，避免误写进度。
- 布局优先使用原生 CSS 弹性换行，不引入运行时尺寸测量和新依赖。
