# 全局图片加载队列设计

## 接口依据

LANraragi 页面通过 `GET /api/archives/{id}/page?path=...` 返回二进制图像。预加载只读取页面，不写入阅读进度。

## 调度规则

- 全局最多 3 个图片读取任务；低优先级任务最多占 2 个槽位，始终为当前页保留 1 个槽位。
- `critical`：当前显示页和双页模式的当前画面。
- `normal`：可见卡片、历史和抽屉缩略图。
- `adjacent`：当前页解码完成后的邻页，按距离及阅读方向稳定入队。
- `preload`：普通后台预热。
- 同一缓存键只执行一次；排队中的预加载若变成当前页，原任务直接提升为 `critical`。
- 存在 `critical` 任务时，不启动新的低优先级任务；已开始的读取不强制中断，以免破坏共享请求或缓存写入。

## Reader 接入

- 当前页明确使用 `critical`。
- 当前页未完成请求和解码时不提交普通邻页预加载。
- 沉浸模式先完成当前页，再启动左右邻页。
- 抽屉缩略图移除局部网络队列，实际请求直接进入全局调度器。

## 保持独立的链路

归档去重页的封面读取用于批量特征计算，并带有 Minion 轮询、重试和 CPU 并发约束，不属于阅读显示链路，继续使用其专用批处理池。
