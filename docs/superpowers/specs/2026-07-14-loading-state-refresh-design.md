# 加载状态与刷新按钮修复设计

## 范围

修复四项问题，不调整阅读器结构、卡片视觉或 LANraragi API：

1. 已缓存邻页切换时短暂闪出加载提示。
2. 阅读器加载提示字体过大。
3. 推荐栏刷新按钮与主页归档刷新按钮不一致。
4. `npm run dev` 首次打开主页时，“全部档案”可能永久停在“正在获取结果...”。

## 根因

### 阅读器提示闪烁

`PageImage` 在新页面开始时立即渲染加载提示。即使目标页已缓存，其异步缓存读取和图片 load/decode 仍需短暂时间，因此产生一帧提示。

### 首次主页加载卡死

开发环境启用 React `StrictMode`，首次 effect 会执行 setup、cleanup、再 setup。首个归档请求在 cleanup 中被取消，但 `doFetch()` 已在请求完成前写入最近请求键和时间。第二次 setup 将其误判为刚完成的重复请求并跳过，首个请求又因序列已失效而不清理 `loading`。

全局图片队列只调度 `getImage()` 和 `primeImage()`，不调度 `lrrApi.search()`，不是直接阻塞源。

## 已选方案

### 主页请求所有权

- 最近请求键和时间只在当前请求成功提交结果后更新。
- 被取消、失败或被新请求替代的请求不参与重复请求抑制。
- 保留 React `StrictMode`、AbortController、请求序列和现有错误提示。

### 阅读器加载提示

- 图片进入 loading 后延迟 160ms 才显示提示。
- 缓存命中或快速解码在延迟内完成时不显示提示。
- error 状态立即显示，不延迟。
- 主标题改为 `clamp(18px, 2.2vw, 28px)`；说明改为 `clamp(13px, 1.4vw, 18px)`；缩小间距。
- 延迟定时器在页面变化和卸载时销毁。

### 刷新按钮

- 推荐栏删除裸 SVG 刷新按钮。
- 复用主页 `.btn` 文字按钮、`6px 12px` 内边距和 `12px` 字号。
- 加载时禁用并显示“刷新中”，空闲显示“刷新”。
- 保留“清理缓存并刷新”说明。

## 验证

- 新增静态/行为检查，覆盖 StrictMode 取消后不得把未完成请求记为新鲜结果、加载提示延迟及定时器清理、刷新按钮统一。
- 运行既有 11 项 Node 回归检查。
- 运行 Vite 生产构建和 `git diff --check`。

## 非目标

- 不移除 React `StrictMode`。
- 不修改全局图片队列并发或优先级。
- 不新增请求超时、重试框架或共享按钮组件。
